Application profile for the OSCAR 1.5 serialization format. Relative to OSCAR
1.4, the current corpus shows broader object support, but `ZZRingElem`,
`QQFieldElem`, and phylogenetic trees still use the older encodings shared with
OSCAR 1.4.
